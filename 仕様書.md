Project: mobile-omniverse
1. 概要 (Overview)
OmniVerse は、複数のSNS（X、Bluesky、Mastodon等）のタイムラインを一つに統合して表示する、次世代の「アグリゲーター型」モバイルブラウザ。各SNSに個別のタブ（セッション）でログインし、それらを裏側で同期・抽出することで、自分だけの最強のフィードを構築する。

2. ビジョン (Vision)
Unified Stream: アプリを切り替えることなく、すべてのSNSの更新を一画面でチェックする。

Anti-Algorithm: プラットフォーム側の推奨アルゴリズムに依存せず、ユーザーが自作の「マージロジック」で情報を整理できる。

No API dependence: API制限に左右されず、ブラウザとして表示されている情報をベースに統合を実現する。

3. 主要機能 (Key Features)
Multi-Session Background Tabs: * サービスごとに完全に分離されたクッキー・ストレージを持つバックグラウンド・タブの保持。

Timeline Scraping Engine:

各SNSのDOM構造を解析し、投稿内容（ユーザー、本文、画像、投稿時間）を抽出するスクリプトの実行。

The "Omni-Feed" UI:

抽出したデータを時系列でソートし、ネイティブアプリのような滑らかさで表示する統合タイムライン。

Cross-Post Action:

一つの入力欄から、複数のSNSへ同時に投稿する機能。

4. 技術スタック (Technical Stack - 暫定)
Framework: Flutter または React Native (iOS/Android共通化のため)

Web Engine: WebView (Headlessモード/バックグラウンド実行)

Logic: TypeScript/JavaScript (DOM抽出ロジック)

Storage: SQLite (抽出した投稿の一時保存・高速検索用)

5. 開発フェーズ (Roadmap)
Phase 1: セッション管理 (MVP)
WebViewを複数立ち上げ、それぞれで異なるSNSにログイン状態を保持できる機能の実装。

Phase 2: データ抽出ロジック
XとBlueskyから投稿データを取得するContent Scriptの作成。

取得したデータを共通フォーマット（JSON）に変換するパーサーの実装。

Phase 3: 統合タイムラインUI
共通フォーマットをリスト表示する「Omni-Feed」画面の実装。

いずれ、iOSにも対応したい。共通部分は共有できるような設計で行きたい
